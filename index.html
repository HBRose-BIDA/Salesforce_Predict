Portfolio Project

Salesforce_Predict

A practical process for identifying which sales opportunities are most likely to close. This project demonstrates data extraction from Salesforce, machine-learning scoring, and delivery in two modes: external report output and optional secure write-back to Salesforce.

Published: hbrose-bida.github.io/Salesforce_Predict/index.html

View Repository Open Project README How It Is Built How It Runs

What This Demonstrates

  • Turning a business pain point into a repeatable analytics process.
  • Using historical won/lost opportunities to train a probability model.
  • Producing ranked output that can guide sales prioritization.
  • Designing for both low-friction reporting and enterprise security needs.
Outcome-focused process, not a one-off script

Project Flow

  1. Extract closed and open opportunity data.
  2. Train model on historical patterns.
  3. Score open opportunities.
  4. Deliver output as file or optional Salesforce write-back.

Primary Deliverable

Ranked scored opportunities are produced as an external output file: data/scored/open_opportunities_scored.csv

This supports review, governance, and reporting without requiring CRM customization.

Optional Advanced Mode

For organizations with strict confidentiality controls, scores can optionally be written back into Salesforce so sensitive forecast data remains inside CRM permissions and access rules.

Optional by design. External output remains the default mode.
